# 📚 Interviews Awesome - 面经解析与分享仓库

这是一个致力于**解析面试真题、沉淀高质量技术面经并进行公开分享**的开源仓库。

本仓库的内容均经过精心梳理与结构化解析，按技术领域与分类路径（如前端、后端、数据库等）有序归档，旨在帮助广大技术求职者攻克面试难关，建立系统化的技术知识体系。

---

## 📁 仓库大纲目录

您可以通过根目录下的 **[outline.md](outline.md)** 文件，快速预览当前仓库收录的所有面经题目。大纲以技术树的形式分类层级呈现，并直接链接到对应的解析文件。

所有的面经解析 Markdown 文件均统一归档在 **`contents/`** 文件夹中。

---

## 🤝 参与贡献与互动流程

我们非常欢迎社区网友共同完善和维护这套面经库！如果您有想要新增的题目，或者发现了现有解析中的错误，请通过 GitHub Issue 提交，我们将第一时间进行审核、解析并发布。

### 1. 提交新面经
如果您遇到了经典的面试题，或者希望我们解析某道题目，请点击 **[New Issue](https://github.com/your-username/interviews-awesome/issues/new)** 并选择或填写以下格式：

* **Issue 标题**：`[新增] <您的面试问题题目>` （例如：`[新增] 解释 JavaScript 闭包的内存泄漏问题`）
* **Issue 内容模版**：
  ```markdown
  ### 📥 面试问题
  请填写具体的问题描述，例如：“解释 JavaScript 闭包的内存泄漏问题以及如何避免”。

  ### 🏷️ 分类建议
  建议的存放路径，例如：“frontend/javascript/memory”

  ### 🏷️ 相关标签
  建议的标签，例如：“javascript, closure, memory-leak”（用逗号分隔）

  ### 💡 提示词/关联问法（可选）
  网友们可能用到的其他问法，例如：“JS闭包泄露”、“闭包内存释放”
  ```

### 2. 反馈与改进建议
如果您在阅读现有面经解析时，发现了描述不准确、代码有 Bug 或有更好的优化思路，请提交纠错 Issue：

* **Issue 标题**：`[建议] <对应面经问题名称>` （例如：`[建议] 解释 JavaScript Event Loop 机制及其工作原理`）
* **Issue 内容模版**：
  ```markdown
  ### 🔗 原文链接
  例如：[解释 JavaScript Event Loop 机制](contents/frontend/javascript/asynchronous/6a4647c06502b4457f2650c0.md)

  ### ✏️ 改进建议/纠错内容
  请在此详细描述您发现的问题，或者您建议修改后的代码与原理解析。
  ```

> 💡 **审核与发布说明**：维护人员在收到 Issue 并审核通过后，会将其录入底层解析系统进行深度结构化解析，随后该解析会自动同步更新至本仓库中。
